# Week 1 Self-Reflection — Milestone 1

## What progress did I make this week?

I created the first working proof-of-concept for the OVS Project Signal Dashboard. The application loads project data from a local JSON file, displays overall dashboard metrics, renders project cards, and allows the user to filter projects by status.

## What is currently working?

The React/Vite application runs locally. The dashboard displays total projects, active projects, average progress, and total blockers. Each project card includes project type, status, priority, task completion, blockers, summary, and progress bars.

## What challenges came up?

The main challenge was keeping the Week 1 scope realistic. I avoided adding larger features such as authentication, a database, or complex APIs because this milestone is focused on proving the interface and core application direction.

## How did GitHub support my workflow?

GitHub is being used to track the project setup, application structure, interface build, data model, documentation, and future changes. My goal is to commit work in stages so the project history shows steady development instead of one final upload.

## How did Trello support my workflow?

Trello is being used to organize course requirements, Week 1 tasks, work in progress, testing/review items, screenshots, and submission preparation. The board mirrors the software production workflow.

## What will I improve next?

For the next iteration, I plan to add stronger data visualization, more interactive project/task details, expanded testing, and a more polished interface based on usability considerations.
